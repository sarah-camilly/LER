## Módulo 2 - Levantamento de Requisitos (Continuação)

- O que é Levantamento de Requisitos
    - Transformar as necessidades do negócio em informações técnicas

- Requisitos Funcionais e Não Funcionais
    - RF - O que o sistema vai fazer
    - RNF - Como o sistema vai fazer

- Requistos, Regras, Restrições
    - Requisi
  stos: A Parte Técnica do sistema
    - Regras: As Necessidade dos Clientes
    - Restrições: Limitações impostas pela, linguagem, leis, normas, 

- Modelo de Documentação de Requisitos

|Campo|Conteúdo|Exemplo|
|----|------|------|
| ID | Código | RF-001 |
| Título | Valor | Tela de Login |
| Descrição | O Ator deve ser Capaz de | O Cliente conseguira fazer Login | 
| Prioridade | Valor da PRioridade | Alta |
| Regras Vinculadas | Outras Regras Relacionada | RN-001 - Banco de Dados |


### Tenicas de levanamentos derequisistos 

- briefing: coleta edados inicias 
    - nome do projeto
    - objetivos do projeto 
    - piblico-alvo 
    - orçamentos 
    - prazos
    - Elemntos de desing (cores, fonte, estilos, logomarca, logotipo, slogan)
        - manual de identidade visual 
    - identidade do cliente
        - missão. visão, valores 
- Brainstorm ou BrainStroming: Técnica de susgestão de ideias para solução de problemas
   - Estruturadas: Rereuniões pautadas, com ordem de fala e tempo pré determinado
        - Não estruturadas: apenas os problemas são apresentados 
        - Mistas: misturam técnicas de fala, de reuniões estruturadas com não estruturadas, podem ter tempo de fala, mas sem ordemde fala.
        - Ferramentas para brainstorm
          - miro, mentimeter, one note, google docs

- questionario:
    - perguntas direcionadas com objeivo de montar o escopo do projeto 
    - nessa etapa o projetoto passa a ganhar corpo e documentação 
    - Exenplos de perguntas:
        - ondeserá acesado?
        - Quem ira acessar? 
        - O uqe acontece quando clica aqui?
    - ferramentas para o questionario:
        - git e github (versioonamento)
        - 
- Entrevista:
    - Ato de conversa com o usário do sistema
    - identificação das dores e conhecimento do usuário 
    - fonte de informação para melhoria do sistema 
    - nessa etapa já existe um MVP (Minímo produto viavél);

- Etnogrfia:
    - tercnica e levantamento de requisitos que consiste em **observar** a cultura do ambiente que ser desenvolvido o projeto. seus valores, seu dia a dia.
    -muitas das vezes uma quipe de desenvolvimento atua dentro da empresa contratante.

- Workshop
    - levar o produto e o seviço proximo  ao stakeHolders (Pessoas interessadas)
    - observar as informações e FeedBacks desses stalkeHolders a fim de melhorar para os próximos lançamentos 
    - lançamentos de produtos em feiras, eventos e workshops de produtos 
    - atigem um público específico (pessoas mais conectadas com os produtos)
